## P0. 정면뷰 ↔ 3D 좌표·크기 일치 검증 (버그 픽스 선행)

피드백 3번("정면뷰 배치와 미리보기가 다르다")의 원인을 기능 추가 전에 규명한다.

- 검증 절차: 폭 100cm 작품을 x=2.00m, 중심높이 150cm, scale 1.0으로 배치 → 3D에서 메시 월드 좌표·바운딩 크기를 콘솔로 실측 → 정면뷰 렌더 값과 대조, 차이를 보고.
- 유력 가설(우선 확인): **정면뷰가 액자·매트를 제외한 그림 크기만 렌더**하고 있어 3D의 액자 포함 외곽과 체감이 다름. 사실이면 정면뷰를 "액자 포함 외곽 기준" 렌더로 수정하고, 그림/액자 경계를 구분 표시.
- 추가 일치 항목: 정면뷰에 룸 조명 무드 틴트 근사 적용(색 체감 차이 축소), 천장 높이·문 개구부·걸레받이 라인 정확 반영.
- 스냅·겹침 판정 로직을 `shared/placementRules.js`로 분리(신설) — P2의 3D 편집과 정면뷰가 **동일 로직을 공유**하게 만들어 이후 불일치를 원천 차단한다.
- AC: 동일 배치값의 정면뷰/3D 실측 오차 리포트 제출(위치·크기 오차 0 목표, 잔여 차이는 사유 명시). 액자 포함 렌더 반영 스크린샷.

## P1. 정면뷰 직접 스케일 핸들 (피드백 2)

- 작품 선택 시 4코너 핸들 표시. 코너 드래그 = 비율 고정 균등 스케일(작품 종횡비는 실측 고정이므로 자유 변형 없음).
- 드래그 중 작품 옆에 실시간 치수 라벨: "92 × 74 cm (scale 1.25)".
- scale 1.0 통과 시 스냅 흡착 + "실측" 틱 표시. Alt(⌥) 드래그 = 미세 조정(스냅 해제).
- 인스펙터 scale 필드와 양방향 동기, undo 스택 통합.
- **회전 핸들은 넣지 않는다**(미술관 작품 특성상 v1 무회전 유지 — 명시적 제외).
- AC: 드래그만으로 크기 조정 → 인스펙터 값 갱신 → undo 복원. 스냅 동작 시연.

## P2. 3D 편집 프리뷰 — "보이는 그대로 조정" (피드백 3, 최대 규모)

정면뷰(정밀 배치)와 공존하는 감각 조정 레이어. **범위: 기존 배치 작품의 위치·스케일 조정 전용.** 신규 걸기·벽 간 이동·삭제는 정면뷰 담당(3D에서는 비활성, 안내 툴팁).

### 동작 방식
- 뷰어를 `?preview=1&edit=1`로 열면 편집 오버레이 활성(데스크톱 전용 — 모바일 프리뷰는 관람만).
- 카메라: 자유 비행(WASD 이동, Q/E 상하, 마우스 드래그 회전, Shift 가속) + [벽 정면 보기] 버튼(선택 작품의 벽 정면으로 카메라 스냅) + [걷기 모드] 토글(아바타 관람 감각 확인용).
- 조작: 클릭 선택 → 외곽 하이라이트 + 카메라를 향하는 4코너 스프라이트 핸들. 몸통 드래그 = 해당 벽 평면 레이캐스트로 x·높이 이동, 코너 드래그 = 균등 스케일. 벽에 아이레벨 150cm 가이드라인 표시.
- 스냅: `shared/placementRules.js` 공용 로직(150cm 흡착, 인접 작품 균등 간격, 겹침·문 간섭 차단) — 정면뷰와 완전 동일해야 한다.

### 데이터 흐름 (에디터가 단일 소스)
- preview → editor postMessage: `{type:'transform', artworkId, placement:{x, centerHeightCm}, scale}` (드래그 종료 시), `{type:'select', artworkId}`.
- editor: state 갱신(undo 1스텝) → 정면뷰·인스펙터 즉시 동기 → 갱신 프로젝트를 preview에 재전송(또는 preview 낙관적 반영 후 확정치 수신).
- AC: ① 3D에서 조정한 값이 정면뷰·인스펙터에 즉시 반영되고 undo로 복원 ② 3D 조정 ↔ 정면뷰 조정 왕복 시 값 손실·튐 없음 ③ 스냅·겹침 규칙이 두 뷰에서 동일 ④ 편집 오버레이는 Publish 산출물에서 완전 제외(엔진 파일에 포함되더라도 edit 파라미터 없이는 코드 경로 미진입, UI 흔적 0).

## P3. 그랜드 로비 (피드백 1)

현행 자동 로비(깊이 4m 고정)를 폐기하고, **편집 가능한 대공간 로비**로 교체한다. 무드 지향: 웅장한 클래식 오페라 하우스 로비 — 단, 특정 건축물·사진의 프레스코/장식을 복제하지 않고 **절차 생성 요소의 조합**으로 웅장함을 만든다.

### 공간
- 기본 크기 w18 × d12 × **h8**(전시룸의 약 2배 층고). 편집 범위 w10–24, d8–18, h5–10.
- 절차 생성 데코(토글, 기본 전부 on):
  - **샹들리에**: 골드 링 토러스 2~3단 + 발광 전구(emissive 구) + 크리스털(소형 Octahedron 다수) + 체인. **실광원은 중앙 1개(PointLight 또는 하향 SpotLight)로 제한** — 전구는 emissive로만 빛나 보이게. 로비 크기에 따라 1~2기.
  - **기둥/필라스터**: 실린더 몸 + 상하 캐피탈(토러스·박스), 좌우 벽면 등간격.
  - **코퍼드 천장**: 골드 빔 격자 + 함몰 패널(웜 톤, 은은한 emissive) — 프레스코 없이 화려함을 담당.
  - **골드 몰딩**: 벽 상단 코니스 + 걸레받이 골드 트림.
  - **레드 카펫**: 입구→전시 입구 문까지 러너.
- 조도: 로비는 밝고 화사한 웜 기본(현행 어두움 해소). 앰비언트 상향 + 샹들리에 광원.

### 기능
- **타이틀 월**: 정면 벽 상부에 전시명 대형 타이포(P4 텍스트 시스템 사용, 기본 세리프·대형·자간 넓게). 그 아래 중앙에 전시 입장 문.
- **작품·포스터 배치**: 로비 4개 벽 모두 정면뷰 편집 대상(대표 포스터, 키비주얼 배치 가능). 문·타이틀 간섭 규칙 동일.
- **에디터 통합**: 하단 룸 스트립 맨 앞에 "로비" 카드(삭제·순서변경 불가, 선택 시 Space 탭에서 크기·벽/바닥·조명·데코 토글 편집).
- 스키마: `museum.json`에 `lobby` 객체 추가 `{ size, wall, floor, lighting, decor:{chandelier, columns, cofferedCeiling, goldTrim, carpet}, artworks:[] }`. **schemaVersion 1 유지** — lobby 필드 부재 시 기본값 자동 생성(기존 프로젝트 호환).
- AC: 로비 크기·데코 편집이 프리뷰에 반영, 스폰 직후 첫 화면(타이틀 월 정면)이 "웅장한 오프닝 샷"으로 성립(스크린샷 보고), 모바일 30fps 유지(라이트 모드: 크리스털 수 감소·emissive 유지).

## P4. 텍스트월 타이포그래피 + 텍스트 조명 (피드백 4)

전시 타이틀월(로비)·섹션 패널에 공통 적용되는 **텍스트월 스타일 시스템**.

### 서체
- vendor 추가: **Noto Serif KR**(명조, OFL) 400·700 KR 서브셋 woff2 — 전시 타이틀의 격을 담당. 기존 Pretendard(산세리프)와 함께 2서체 체제. `NOTICE.md` 라이선스 갱신.

### 스타일 필드 (제목 블록·본문 블록 각각)
```jsonc
"style": {
  "font": "serif" | "sans",
  "weight": 400 | 700,
  "sizeCm": 24,            // 벽면 실측 글자 높이
  "color": "#F2E9DB",
  "letterSpacing": 0.05,   // em
  "lineHeight": 1.6
}
// 패널 공통: align("left"|"center"), bg("none"|"light"|"dark"), widthCm,
//            light: { on, intensity 0.5–2.0, temp "warm"|"neutral"|"cool" }
```
- `bg:"none"` = 벽면 직접 인쇄 모드(투명 배경 canvas 텍스처) — 레퍼런스 사진처럼 벽 위에 타이포+조명만. "light"/"dark" = 판넬 배경.
- **텍스트 스포트라이트**: 작품 조명과 동일한 SpotLight 시스템을 텍스트월에 재사용(타원형 웜 워시가 텍스트를 감싸는 연출). artwork.js의 조명 생성 함수를 공용화.
- 렌더: 고해상 canvas(장변 2048px) → 텍스처. 자간은 `ctx.letterSpacing` 사용하되 **미지원 브라우저 대비 글자 단위 수동 렌더 폴백**(지원 여부 실측 확인 — 작업 원칙 3).
- 에디터: 텍스트월(로비 타이틀·섹션 패널) 선택 시 인스펙터에 타이포 컨트롤 전체 노출 + 정면뷰에 텍스트월 표시·선택 가능화. 실시간 반영.
- AC: 서체/크기/굵기/색/자간/행간/정렬/배경 모드 변경이 프리뷰 즉시 반영, `bg:"none"`+스포트라이트 조합이 레퍼런스 무드 재현(비교 스크린샷), 한글 줄바꿈 자연스러움(단어 단위).

## P5. 벽·바닥 자유 색 + 패턴 확장 + 이미지 업로드 (피드백 5)

- **벽 색**: 프리셋 4종 → **자유 컬러 피커**(HSV 팔레트 + HEX 입력)로 확장. 프리셋 4종은 피커 하단 퀵 스와치로 유지.
- **패턴 프리셋 확장**(절차 생성, 벽 색에 톤온톤 자동 틴트): 다마스크(기존) / 세로 잔줄무늬 / 플라스터 노이즈 / 패브릭 헤링본 / 도트 격자 / 민무늬. 패턴 강도 슬라이더(0–100%).
- **커스텀 패턴 업로드**(벽·바닥 공통): 이미지 업로드 → 장변 1024px 리사이즈·WebP(기존 파이프라인 재사용) → 타일링 텍스처 적용. 컨트롤: 반복 크기 슬라이더(0.25m–4m) + **미러 반복 토글**(THREE.MirroredRepeatWrapping — 비이음매 이미지의 경계 티 감소).
- 바닥: 우드 프리셋 4종 유지 + 커스텀 업로드 슬롯.
- 스키마: `wall: { color, pattern: "damask"|…|"custom", patternAsset?, patternScale, patternOpacity }`, `floor: { preset | "custom", asset?, scale }`. 구 프리셋 → 신 필드 자동 매핑(deep-red → #7A1F2B 등, 로더에서 변환).
- Publish: 업로드 패턴을 `assets/patterns/`로 포함. export README에 한 줄 추가: "업로드한 패턴·이미지의 저작권 확인은 제작자 책임".
- AC: 임의 HEX 벽색 + 업로드 패턴 + 반복 크기 조정이 정면뷰·프리뷰·Publish 산출물 모두에서 동일하게 렌더. 기존 프로젝트(구 프리셋) 로드 시 외관 변화 없음.

### 화면 구성
- 부팅 → 로딩 → **풀스크린 캐릭터 선택**(별도 시작 패널 없음). 상단: 전시 타이틀(meta.title, P4 세리프 스타일). 중앙: **4캐릭터 실시간 3D 라인업**(정면, 유휴 숨쉬기 애니메이션, 바닥 소프트 섀도 블롭, 어두운 웜 그라데이션 배경 + 은은한 골드 파티클). 각 캐릭터 위에 이름 라벨만 표기 — 레벨·아이콘·기타 게임 UI 요소 없음. **레퍼런스 이미지는 무드 참고일 뿐, 특정 게임의 UI 레이아웃·장식 요소를 모사하지 않는다.**
- 선택: 클릭/탭 → 선택 캐릭터 5% 확대 + 웜 림 글로우 + 소량 반짝 파티클(canvas 스프라이트, ≤60개), 나머지는 살짝 딤. 하단에 의상 색 스와치 + 몸 변형 토글(v1.1 커스텀 슬롯 유지, 컴팩트 배치).
- [입장하기] → **조작법 팝업**(모달): WASD·Shift·드래그·E·H·T·P 표. 터치 환경 감지 시 모바일 조작 문구로 자동 교체. [시작] 클릭 시 입장. 팝업 "다시 보지 않기" 체크(localStorage — 배포 사이트는 일반 웹이므로 사용 가능).
- 모바일: 라인업 가로 스와이프/탭 선택, 4체 동시 렌더 부담 시 라이트 모드에서 pixelRatio 하향으로 대응.
- AC: 4종 선택·이름 표기·이펙트·조작법 팝업·커스텀 슬롯 동작, 선택→입장 흐름이 데스크톱/모바일 모두 매끄러움(영상 또는 GIF 보고), avatarDefaults의 preset이 초기 선택 상태로 반영.
