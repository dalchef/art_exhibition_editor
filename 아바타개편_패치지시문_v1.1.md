# 아바타 개편 패치 지시문 v1.1 — 동물 캐릭터 4종

> - 본 문서는 `미술관메이커_개발지시문_v1.md`의 **§5.2(아바타)와 §7 중 아바타 관련 항목을 대체**하는 증분 패치다. 그 외 모든 조항(자기완결 원칙, 절차 생성, 외부 에셋 금지, 작업 원칙 0장)은 그대로 유효하다.
> - 정본 디자인: `docs/character-sheets/` 폴더의 개별 턴어라운드 시트 4장(HEX 팔레트 포함). 라인업 이미지는 참고용이며 색이 다를 경우 **개별 시트가 우선**한다.
> - 작업 성격: 단일 패치(마일스톤 분할 없음). 완료 기준은 §8.

---

## 1. 목표

기존 avatar.js의 캐릭터를 폐기하고, 시트 기반 동물 캐릭터 4종(부엉이·카피바라·토끼·용)을 **코드 절차 생성**으로 재구현한다. 외부 3D 모델 파일(GLB 등)·외부 텍스처 사용 금지. 시트는 디자인 정답지이며 리포에 이미지로 포함되어 있으니, 구현 중 반드시 이미지를 열어 실루엣·배색을 대조할 것.

- `docs/character-sheets/owl.png` — 부엉이 (조끼)
- `docs/character-sheets/capybara.png` — 카피바라 (목도리)
- `docs/character-sheets/rabbit.png` — 토끼 (네커치프)
- `docs/character-sheets/dragon.png` — 용 (조끼)
- `docs/character-sheets/lineup.png` — 참고용 라인업

## 2. 파일 구성

- `viewer/js/avatar.js` — 전면 재작성: 공통 빌더 + 애니메이션.
- `viewer/js/avatarPresets.js` — 신설: 캐릭터별 설정 상수(부위 파라미터·색 테이블·변형 색 세트). 형태 데이터와 로직을 분리한다.
- 시작 화면(hud.js) 캐릭터 선택 UI 갱신(§6).

## 3. 공통 시스템

### 3.1 좌표·치수 기준
- 발바닥 y=0, 캐릭터 정면 +Z. 단위 m.
- 기준 신장: **정수리 1.30m** (귀·뿔·귀깃 제외. 부속물 초과 허용 — 토끼 귀 끝 ~1.7m 무방).
- 충돌 캡슐: 4종 공통 **반경 0.40m / 높이 1.30m** (v1.0의 0.35/1.4를 대체 — 통통 체형 반영). 카메라 추적 타깃 높이 1.0m 공통.
- §5의 모든 수치는 **시작값**이다. 시트와 나란히 놓고 눈으로 대조하며 ±20% 범위에서 미세 조정을 허용한다. 단 **부위 구성·개수·HEX는 고정**.

### 3.2 재질
- 전 부위 `MeshToonMaterial` + 3단 gradientMap(기존 공통 유틸 재사용), 무광.
- 눈: 무광 검정 구(#2B2B2B), 하이라이트 없음 — 시트의 "점 눈"이 스타일 핵심.
- 재질 인스턴스는 색상별 공유(의상 색 변경 시 해당 재질 1개만 갱신되도록).

### 3.3 지오메트리 원칙
- 허용 프리미티브: Sphere, Capsule, Cylinder(테이퍼), Cone, Torus, Box, Plane. 스케일 변형으로 달걀·주둥이·날개 등을 만든다. addons(examples/jsm) 사용 금지 유지.
- **각진 로우폴리 금지** — 클레이 토이의 매끈함이 정체성. 세그먼트: 대형 구 28×20 이상, 소형 부위 16×12. 예산: 캐릭터당 삼각형 ≤ 20k, 드로우콜 ≤ 40 (동시 표시 아바타는 1체뿐이므로 여유 있음).
- 몸 유형 2종:
  - **일체형**(부엉이·카피바라): 머리+몸 = 달걀 1덩어리, 이목구비를 표면에 부착.
  - **분리형**(토끼·용): 머리 구 + 몸 달걀, 목 겹침 ~0.08m.

### 3.4 얼굴·라인 데칼 유틸
- `makeFaceDecal(drawFn)` 공통 유틸: 512px 투명 canvas에 코·입·콧구멍·배 라인을 그려 CanvasTexture로 만들고, 소형 Plane에 입혀 표면에서 0.005m 띄워 부착(transparent, depthWrite=false, polygonOffset로 z-fight 방지).
- 선 색: 다크 브라운 #5B4636, 둥근 획(lineCap round), 시트의 손그림 느낌 유지.
- 사용처: 토끼(코 채움 #F1C9B6 + w자 입), 용(스마일 + 콧구멍 2점), 카피바라(주둥이의 세로선+양갈래 입), 용 배(가로 라인 4개, #E4D3AC).

## 6. 커스터마이즈 & 스키마 변경

### 6.1 커스텀 슬롯 (v1.0의 "머리색/옷색"을 대체)
- **의상 색**: 팔레트 8색 자유 선택 — #D97A5C, #8DA98A, #D9B35A, #F2E9DB, #7A8FA6, #C98A8A, #5B5B5B, #A9744F.
- **몸 색 변형**: 동물별 큐레이션 2~3종. 변형은 단일 색이 아니라 **파생색 세트**(몸 + 주둥이/내이/배/발 등)로 정의한다. 기본 세트는 §4, 추가 변형 세트(부엉이 그레이·스노위 / 카피바라 다크탠 / 토끼 그레이·브라운 / 용 더스티블루·라일락)는 구현자가 상수 테이블로 제안하고 **스와치 스크린샷 보고 → 승인 후 확정**한다. 임의 자동 산출 금지.

### 6.2 museum.json — avatarDefaults 필드 교체 (schemaVersion 1 유지, 배포 데이터 없음)
```jsonc
"avatarDefaults": { "preset": "capybara", "body": "default", "garment": "#8DA98A" }
// preset: "owl" | "capybara" | "rabbit" | "dragon"
```
- `shared/schema.js` 기본값·검증, 에디터 state.js, 샘플 museum.json 모두 갱신.

### 6.3 시작 화면
- 캐릭터 카드 4개 + 선택 캐릭터의 느린 턴테이블 실시간 렌더 1개 + 의상 색 팔레트 + 몸 변형 토글.
- 카드 아이콘도 외부 이미지 없이: 부팅 시 오프스크린 렌더로 4종 스냅샷을 1회 생성해 사용(자기완결 원칙 유지).

## 7. 애니메이션 (공통, avatar.js)

- 걷기: y 바운스 ±0.03m(보행 속도 비례 주기) + 좌우 롤 ±3~4°(일체형은 이 뒤뚱거림이 곧 걸음) + 팔 스윙 ±15°(분리형·부엉이 날개팔).
- 부속물 관성: 토끼 귀, 카피바라 목도리 앞자락, 용 꼬리 끝 — 이동 가감속에 위상 지연 ±5° 회전. 캐릭터 생기의 핵심이므로 생략 금지.
- 유휴: 숨쉬기 스케일 ±1%(2.5초 주기), 8초마다 미세 좌우 둘러보기.
- 이동 방향 회전 스무딩은 기존 controls.js 로직 유지.

## 8. 완료 기준 (AC)

1. 시작 화면에서 4종 선택·의상 색·몸 변형이 실시간 반영된다.
2. **시트 대조 보고**: 캐릭터마다 정면·측면·후면 인게임 스크린샷을 해당 시트와 나란히 배치한 비교 이미지를 제출하고, 실루엣·배색 일치를 확인받는다.
3. 걷기·유휴·부속물 관성 애니메이션이 §7대로 동작한다.
4. 성능: 캐릭터당 삼각형 ≤ 20k, 아바타 드로우콜 ≤ 40, 모바일 30fps 유지(M2 기준 재확인).
5. 외부 에셋·addons 사용 0건. 시트 PNG는 docs/ 참고용일 뿐 런타임에 로드되지 않는다.
6. Publish ZIP 산출물에서 새 아바타가 정상 동작(자기완결 회귀 확인).
