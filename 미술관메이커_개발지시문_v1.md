## 0. 구현 담당 작업 원칙 (전 과정 필수 준수)

1. 여러 단계가 필요한 작업은 시작 전 계획을 세우고, 마일스톤 단위로 검증한 뒤 다음으로 진행한다.
2. 완료 기준은 "코드를 작성했다"가 아니라 **실제로 동작하고 검증된 상태**다. 시작한 작업은 끝까지 완료한다.
3. 존재하지 않는 API·라이브러리·함수를 추측으로 사용하지 않는다. 공식 문서 또는 실제 응답으로 확인한 뒤 사용하고, 확인하지 못했으면 그 사실을 보고한다. (특히 Three.js API, Met/AIC API의 실제 응답 스키마는 반드시 실측 확인)
4. 파일 생성이 필요한 작업은 실제 파일을 만들고 경로를 보고한다. 채팅에 코드만 출력하고 끝내지 않는다.
5. 막히면 조용히 넘어가지 않는다. 보고 형식: 지금까지의 결과 + 막힌 지점 + 시도한 방법 + 대안 제시.
6. 시크릿 하드코딩 금지. (v1은 API 키가 불필요한 공개 API만 사용하므로 시크릿 자체가 없어야 정상)
7. 본 문서와 충돌하는 판단이 필요하면 임의로 변경하지 말고, 옵션을 정리해 질문한다.

## 1. 제품 정의

퍼블릭 도메인 미술 작품 이미지로 3D 전시 공간을 직접 큐레이팅(**에디터**)하고, 완성본을 **미술관별 독립 정적 사이트**로 내보내 누구나 아바타로 걸어다니며 관람(**뷰어**)할 수 있는 웹 도구.

### 사용 시나리오 (제품이 복무해야 할 흐름)

1. 내부 제작자 1인이 에디터에서 전시 1개(= 미술관 1개)를 제작한다. 구성: 섹션 3~4개(= 룸), 작품 총 12~15점, 섹션당 3~4점.
2. 도슨트가 뷰어의 **녹화 모드**로 플레이 → 화면 녹화 → 유튜브 콘텐츠(약 1시간)를 제작한다.
3. 콘텐츠 공개 후, 미술관을 ZIP으로 내보내 GitHub Pages 또는 자체 웹 주소에 업로드 → 구독자가 링크로 방문해 직접 관람한다.

**도슨트 해설 기능(오디오·자막·설명 팝업 낭독)은 소프트웨어에 없다.** 해설은 사람이 한다. 소프트웨어는 무대만 제공한다.

### 좌표·배치 규칙 (에디터/뷰어 공통, 문서화 필수)

- 평면도 기준 방위: 화면 위 = `north`, 오른쪽 = `east`. Three.js 축 매핑은 구현 재량이되 `world.js` 상단에 주석으로 고정 문서화.
- 정면뷰 `x`: 해당 벽을 **룸 내부에서 바라볼 때** 왼쪽 끝(0)에서 작품 중심까지의 거리(m).
- `centerHeightCm`: 바닥에서 작품 중심까지. 기본값 150 (미술관 아이레벨 표준).
- 문: 폭 1.8m × 높이 2.6m 고정. `offset` = 벽 왼쪽 끝에서 문 중심까지(m).
- 룸 자동 배치: room[0]부터 순서대로, `exitDoor.wall` 방향으로 다음 룸을 이어 붙인다(north = 직진, east/west = 회전). 룸 사각형이 겹치는 구성은 에디터가 저장 전 검증하여 차단하고 사유를 표시한다.
- 스폰: room[0] 앞에 **자동 생성 로비**(폭 = room[0] 폭, 깊이 4m). 로비 정면 벽 = 전시 타이틀월(meta.title / subtitle / curator / intro). 아바타는 로비 입구에서 타이틀월을 바라보며 시작 → 첫 화면이 곧 오프닝 샷이 된다.
- 섹션 서문 패널: 각 룸 진입 문에서 가장 먼저 보이는 벽에 자동 배치(v1은 자동 위치 고정).
- **`docentNote`는 Publish export 시 museum.json에서 반드시 제거**한다. 비공개 메모가 공개 사이트에 노출되면 안 된다. (프로젝트 저장/프리뷰에는 유지)

## 5. 뷰어(관람 모드) 기능 명세

### 5.1 월드 구성
- 룸: 벽(두께 0.2m 박스), 바닥, 천장, 문 개구부, 걸레받이·간단 몰딩.
- 벽지·바닥 텍스처: **전부 코드로 절차 생성**(canvas 타일링 — 다마스크풍 모티프, 헤링본 우드 등). 외부 텍스처 파일 다운로드 금지(저작권 리스크 0 원칙).
- 데코: 벤치(충돌 있음), 천장 트랙조명 소품(장식), 룸별 토글 반영.
- 타이틀월·서문 패널: canvas 텍스처 텍스트, Pretendard 사용(FontFace 로드 완료 후 렌더).

### 5.2 아바타
- 오리지널 로우폴리 절차 생성: 둥근 캡슐 몸 + 구체 머리 + 머리카락 메시 변형 몇 종. `MeshToonMaterial` + gradientMap으로 톤 셰이딩. 키 약 1.4m(2.5등신 귀여운 비율).
- 프리셋 4종(머리 스타일 2 × 실루엣 2) + 머리색/옷색 커스텀.
- 걷기 연출: 절차적 바운스 + 이동 방향 회전 스무딩. 외부 3D 모델 파일 의존 금지(자기완결 export를 위해).
- 시작 화면: 전시 타이틀 + 아바타 프리셋/색 선택 + 조작법 안내 → [입장]. `avatarDefaults`가 초기값.

### 5.3 조작
- 데스크톱: WASD/화살표 이동(2.2m/s), Shift 달리기(3.6m/s), E = 상호작용, ESC = 닫기.
- 카메라: 3인칭 후방 추적(거리 ~4.2m, 높이 ~2.4m), 위치·회전 모두 댐핑(부드러운 팔로우가 녹화 품질의 핵심). 벽 근접 시 레이캐스트로 카메라 당겨오기(벽 뚫림 방지).
- 모바일: 좌측 가상 조이스틱(포인터 이벤트로 **자체 구현**, 외부 라이브러리 금지), 상호작용은 화면 프롬프트 버튼 탭.
- 충돌: 벽·벤치 대상 단순 원형(반경 0.35m) vs AABB 판정. 문 개구부 통과 가능.

### 5.4 작품 상호작용 — 줌 모드
- 근접 조건: 작품과 2.2m 이내 + 시선 방향 ±35° → "E / 자세히 보기" 프롬프트.
- 줌 모드 = **HTML 풀스크린 오버레이**(3D 아님 — 화질과 텍스트 선명도 우선): 2048px 최적화 이미지 + 캡션 패널(제목/작가/연도/재료/실측 크기/소장처/크레딧).
- Ken Burns 토글: 느린 팬·줌 애니메이션 on/off (녹화 시 디테일 설명 구간용).
- ESC/탭으로 복귀. 오버레이 열림 중 이동 입력 차단.

### 5.5 명제판
- 각 작품 옆 벽면에 소형 명제판(제목 + 작가, canvas 텍스처, Pretendard). 실제 미술관의 라벨 감성. 상세 정보는 줌 모드에서.

### 5.6 녹화 모드
- `H`: 모든 HUD(프롬프트·조이스틱·버튼) 표시/숨김 토글. 숨김 상태에서 화면에는 3D 월드만 남아야 한다.
- `T`: 시계·날짜 오버레이 토글(실시간, "PM 4:17 / 7월 5일" 형식, 좌하단 라운드 박스 — 레퍼런스 무드의 오리지널 디자인).
- 녹화 모드는 별도 상태가 아니라 위 토글 조합으로 성립. 조작법 안내에 명기.

### 5.7 오토 도슨트 워크 (M7 스트레치)
- `P`: `route` 순서대로 자동 보행. 각 작품 앞 정지점에서 기본 4초 대기(`?dwell=초` URL 파라미터로 조정), 작품을 바라보게 회전. 룸 간 이동은 문 중심 경유 직선 경로(선형 구조라 경로탐색 불필요). 아무 키나 누르면 해제.

### 5.8 성능·로딩
- 목표: 데스크톱 60fps / 모바일 30fps.
- 모바일 라이트 모드(자동 감지): pixelRatio ≤ 1.5, 그림자 off, 활성 조명 수 제한(현재 룸만).
- 로딩 화면: 전시 타이틀 + 진행률. 에셋은 룸 단위 우선순위 로드 허용.
- 텍스처 예산: 작품당 장변 2048px, 전체 에셋 40MB 이하 권장.

## 6. 에디터 기능 명세

### 6.1 레이아웃 (목업 준수)
- 상단바: 로고/전시 제목, 실행취소·다시실행, [Preview], [Publish].
- 좌측 패널: **Artwork 탭**(업로드 / My Artworks 그리드 / 선택 작품 속성) · **Space 탭**(선택 룸 속성).
- 중앙 작업영역: 평면도 모드 ↔ 정면뷰 모드 전환.
- 우측 패널: Space & Atmosphere — 벽 색/패턴 4종 스와치, 바닥 4종, 조명 무드 3종(Warm/Neutral/Cool), 데코 토글(Benches/Spotlights). 선택된 룸에 적용.
- 하단: 룸 스트립(썸네일 + 섹션명) + [Add Room]. 드래그로 순서 변경(= 동선·문 연결 자동 갱신).

### 6.2 프로젝트·저장
- 자동저장: 상태 변경 후 2초 디바운스로 IndexedDB에 저장(JSON + 이미지 Blob 분리 저장). 새로고침 후 완전 복원.
- 프로젝트 파일: `.zip` 내보내기/불러오기(museum.json + assets). NAS 백업용. Publish ZIP과 구분되는 "작업 파일"임을 UI에 명시.
- Undo/Redo: JSON 상태 스냅샷 방식(이미지 Blob은 id 참조로 제외), 최대 50단계.

### 6.3 업로드 파이프라인
- PNG/JPG/WEBP 드롭 또는 선택 → canvas로 장변 2048px 리사이즈 → WebP(품질 0.85) 인코딩(브라우저 미지원 시 JPEG 폴백) + 256px 썸네일 생성 → IndexedDB 저장 → My Artworks 그리드에 표시.
- 업로드 직후 실측 크기(cm) 입력을 요구(비율은 이미지에서 자동 계산해 한 변만 입력해도 되게).
- 8000px급 대형 원본도 브라우저가 죽지 않고 처리되어야 함(`createImageBitmap` 활용).

### 6.4 평면도 모드
- 룸 사각형을 선형 배열로 표시. 룸 크기 드래그 + 수치 입력. 문 위치(offset) 드래그.
- 룸 순서 변경 시 문 연결 자동 재계산, 겹침 구성은 차단 + 사유 표시(§4 규칙).
- 배치된 작품은 벽 위 마커로 표시(전체 조망용). **벽을 클릭하면 해당 벽의 정면뷰 모드로 진입.**

### 6.5 정면(입면) 뷰 모드 — 작품 배치의 중심
- 선택한 벽을 실척 2D로 렌더(벽지 프리셋 반영, 바닥선·문 개구부 표시).
- My Artworks에서 드래그하여 걸기. 선택 시 이동 핸들 + 삭제/복제 툴바(목업과 동일한 조작감).
- 실측 크기로 표시. 스냅 가이드: 중심높이 150cm 수평선, 작품 간 균등 간격, 상단/중심 정렬선.
- 작품 간 겹침, 문 개구부 간섭 방지(스냅으로 밀어냄 + 경고).

### 6.6 인스펙터 (작품 선택 시, 좌측 패널)
- 캡션 전체 필드(제목/작가/연도/재료/소장처/크레딧/출처 URL) + [API로 채우기] 버튼.
- 실측 cm(폭·높이), scale, 액자 스타일 4종 + 매트 on/off·색, 스포트라이트 세기(0.5–2.0)·색온도.
- 관람 순서: route 내 위치 표시 + 순서 변경 UI(전체 route 리스트 패널 포함 — 이 리스트가 곧 대본 목차).
- 도슨트 메모: 여러 줄 텍스트. **"비공개 — 게시 시 제거됨" 라벨을 항상 표시.**

### 6.7 API 캡션 자동 채움
- 검색창(제목/작가) → Met + AIC 병렬 검색 → 결과 카드(썸네일·제목·작가·연도·퍼블릭 도메인 여부) → 선택 시 캡션 필드 채움(모든 필드 수동 수정 가능).
- 사용 API (둘 다 키 불필요, 구현 전 실제 응답을 curl로 확인하고 필드 매핑을 확정할 것):
  - Met: `https://collectionapi.metmuseum.org/public/collection/v1/search?hasImages=true&q={q}` → objectIDs → `/objects/{id}` (title, artistDisplayName, objectDate, medium, dimensions, primaryImage, isPublicDomain)
  - AIC: `https://api.artic.edu/api/v1/artworks/search?q={q}&fields=id,title,artist_display,date_display,medium_display,dimensions,image_id,is_public_domain` / 이미지: IIIF `https://www.artic.edu/iiif/2/{image_id}/full/843,/0/default.jpg`
- 치수 문자열 파싱(예: "73.7 × 92.1 cm")은 best-effort. 실패 시 수동 입력 유도.
- 이미지 직접 가져오기: 각 API 이미지 서버의 CORS를 실측 확인 후 가능한 경우에만 [이미지 가져오기] 제공. 불가하면 "출처에서 다운로드 후 업로드" 안내로 대체.
- API 장애·미응답이 수동 입력 흐름을 절대 막지 않아야 한다.

### 6.8 Preview
- [Preview] → 뷰어를 새 창(동일 출처)으로 열고 postMessage 핸드셰이크로 현재 프로젝트 JSON + 이미지 blob URL 전달. 뷰어는 `?preview=1`일 때 이 경로로 데이터 수신(기본 경로: `fetch('./data/museum.json')`).
- 프리뷰에서 보이는 것 = 관람자가 보게 될 것과 동일해야 한다(docentNote만 예외적으로 유지).

### 6.9 Publish (export)
- 슬러그 확인 → ZIP 생성:
```
{slug}/
├── index.html            # 뷰어. OG 메타 자동 삽입(og:title, og:description=서문 요약, og:image=대표작 썸네일)
├── viewer/ …             # manifest.json 기준으로 엔진 파일 복사
├── vendor/ …             # three, pretendard (jszip 제외 — 뷰어에 불필요)
├── data/museum.json      # docentNote 제거본
├── assets/artworks/*.webp
├── assets/thumbs/*.webp
├── NOTICE.md
└── README.md             # 한국어: ① GitHub Pages 배포(새 repo → 업로드 → Settings→Pages→main/root), ② 자체 호스팅(폴더 업로드), ③ 로컬 확인법
```
- 압축 해제 후 아무 정적 호스팅에 올리면 즉시 동작하는 완전 자기완결 번들이어야 한다(외부 CDN 의존 금지).

## 7. 비주얼·연출 가이드

- 무드: 따뜻한 클래식 갤러리. 벽 프리셋 = 딥레드(다마스크풍 패턴)/딥그린/네이비/웜그레이, 바닥 = 헤링본 우드 4톤. 전부 절차 생성.
- 조명: 룸 무드(warm #ffd9a8 계열 / neutral / cool) 앰비언트+헤미스피어, 작품별 SpotLight. 그림자는 데스크톱만(성능).
- 액자 4종: `gold`(밝은 골드 이중 몰딩 프로파일), `wood`(내추럴), `black`(모던 슬림), `none`. 프로파일은 절차 지오메트리로 생성. 매트 = 크림색 인셋 평면.
- 폰트: Pretendard 전면 사용(HUD/줌/명제판/타이틀월). vendored woff2, 시스템 폰트 폴백.
- 전체적으로 "아늑한 토이 디오라마 미술관" — 둥근 실루엣, 채도 낮춘 따뜻한 색. 특정 게임의 에셋·캐릭터·UI를 모사하지 않는다.

## 8. 마일스톤 & 완료 기준 (순서 고정 — 뷰어 먼저: 3D 보행 감각이 최대 리스크이므로 최우선 검증)

**M0 — 골격**: 저장소 구조, three vendoring(버전 기록), `shared/schema.js`, 샘플 museum.json + 퍼블릭 도메인 작품 3점, README.
- AC: 로컬 서버로 viewer가 빈 룸을 렌더한다.

**M1 — 뷰어 프로토타입**: 룸 1개(벽지/바닥/조명), 아바타 보행+충돌+추적 카메라, 작품 3점(액자+명제판+스포트라이트), 근접 프롬프트+줌 모드.
- AC: 데스크톱 크롬 60fps로 걷고, 줌 모드에서 캡션이 표시된다.

**M2 — 뷰어 완성**: 다중 룸+문 통행, 로비 타이틀월+섹션 서문 패널, 시작 화면(아바타 선택), 모바일 조이스틱+라이트 모드, 녹화 토글(H/T), 로딩 화면.
- AC: 모바일 브라우저 30fps 관람 가능. H를 누르면 화면에 3D 월드 외 아무것도 남지 않는다.

**M3 — 에디터 코어**: 상태/자동저장/undo·redo, 업로드 최적화 파이프라인, 평면도 모드, 정면뷰 배치(드래그/핸들/스냅/겹침방지), 인스펙터 전체, route 관리.
- AC: 15점 배치 프로젝트가 새로고침 후 완전 복원되고, 프로젝트 zip 내보내기/불러오기가 왕복 무손실이다.

**M4 — 공간 무드 + 프리뷰**: 우측 패널(벽/바닥/조명/데코) 적용, Preview 연동.
- AC: 벽 색 변경이 프리뷰에 반영된다.

**M5 — API 캡션**: Met/AIC 검색·자동 채움.
- AC: "Starry Night" 검색으로 고흐 캡션이 채워진다(실제 API 응답으로 검증). API 차단 상태에서도 수동 입력이 정상 동작한다.

**M6 — Publish**: ZIP export → GitHub Pages 실배포로 검증.
- AC: 배포 URL에서 모바일 관람이 성공하고, 배포본 museum.json에 docentNote가 존재하지 않는다.

**M7 — 스트레치**: 오토 도슨트 워크(P), 아바타 프리셋 확장, 프리뷰 내 벽 바로가기.

### 공통 테스트 체크리스트
크롬·사파리·모바일 각 1회 / 새로고침 복원 / 8000px 대형 이미지 업로드 / 한글 캡션·명제판 렌더(폰트 로드 타이밍 포함) / 룸 겹침 구성 차단 / `file://` 안내 문구 정확성 / 배포본 외부 네트워크 요청 0건.
