## P1. 스플릿 3D 라이브 프리뷰 (에디터)

중앙 캔버스를 좌우 분할한다. 왼쪽은 기존 편집 뷰(평면도/정면뷰 탭 유지), 오른쪽은 3D 라이브 프리뷰.

- 분할 경계는 드래그로 비율 조절. 프리뷰 패널 접기/펼치기 토글 제공(접으면 기존 전체폭 편집 화면, 렌더 루프 일시정지).
- 기존 미리보기 창의 뷰어를 임베드 모드로 재사용한다. 임베드 모드 플래그(예: `?embed=1&skipIntro=1`)에서는 캐릭터 선택 화면을 건너뛰고 기본(또는 마지막 선택) 아바타로 즉시 갤러리에 진입한 상태로 시작. **퍼블리시 결과물의 정상 시작 흐름(캐릭터 선택)에는 영향 없어야 함.**
- 기존 "프리뷰 창 즉시 반영" 동기화 로직을 재사용해 편집 변경(작품, 텍스트, 벽, 바닥, 조명, 룸 구조)을 실시간 반영. 연속 입력(슬라이더)은 300ms 디바운스. 구조 변경(룸 추가/삭제/이동)은 씬 재구성으로 처리 가능.
- (2차, 여유 시) 편집 뷰에서 선택 중인 룸/벽으로 3D 카메라 자동 이동.

**수용 기준:** 벽 색 변경 시 1초 내 프리뷰 반영. 캐릭터 선택 화면이 프리뷰에 나타나지 않음. 퍼블리시 결과물은 기존대로 캐릭터 선택부터 시작.

## P2. 평면도 상호작용 개편 (에디터)

### 2-1. 룸 선택 → 벽 호버 스냅 → 클릭 진입 (단계적 선택)

현재 문제: 룸 클릭 시 곧바로 벽 설정(정면뷰)으로 진입해버림.

1. **룸 선택:** 룸 클릭 시 해당 룸이 활성화되고 시각적 강조(외곽선/배경 하이라이트). 화면 전환 없음. 우측 패널은 해당 룸의 공간 설정 표시.
2. **벽 호버 스냅:** 활성 룸에서 커서가 벽 근처(화면 기준 약 12px 이내)로 가면 가장 가까운 벽 세그먼트가 자석처럼 스냅되어 하이라이트(색상 강조 + 두께 증가). 클릭 전에 어느 벽이 선택될지 명확히 보여야 함.
3. **벽 진입:** 하이라이트된 벽 클릭 시 정면뷰로 진입.
- 빈 공간 클릭 또는 ESC로 선택 해제/뒤로가기.

### 2-2. 룸 드래그 이동 + 크기 조절

현재 문제: 문 방향(동서남북)을 먼저 정해야 룸 배치(일직선/ㄱ자 등)가 결정되는 역순 방식이라 직관적이지 않음.

- **룸 이동:** 활성 룸을 드래그해 평면도 상에서 자유롭게 이동. 다른 룸의 벽에 가까워지면 자석 스냅으로 딱 붙음(겹침 방지). 룸을 원하는 모양으로 배치한 뒤 문을 정하는 순방향 흐름.
- **크기 조절:** 활성 룸의 모서리/변에 핸들 표시, 드래그로 크기 조절. 조절 중 치수(m) 실시간 표시. 기존 수치 입력 방식도 유지.
- **문 위치 자유화:** 맞닿은 벽(또는 개구부가 가능한 벽) 위에서 문 핸들을 드래그해 좌우 위치 이동. 룸 재배치 시 문이 유효하지 않은 위치(맞닿지 않는 벽 등)가 되면 경고 표시. 기존 문 핸들 드래그 기능이 있다면 새 배치 방식과 통합.
- **배치물 보호:** 벽 길이가 줄어 작품/텍스트가 벽 범위를 벗어나면 해당 오브젝트를 빨간 하이라이트로 경고. 자동 삭제 금지.

**수용 기준:** 룸 클릭 → 강조만 됨(화면 전환 없음) → 벽 호버 시 해당 벽만 하이라이트 → 클릭 시 정면뷰 진입. 룸 두 개를 드래그로 ㄱ자 배치 후 맞닿은 벽에 문 배치 가능. 룸 축소로 작품이 벽 밖으로 밀리면 경고 표시되고 삭제되지 않음.

## P3. 벽면(face) 단위 스타일 분리 — 데이터 모델 변경 (에디터 + 뷰어)

현재 문제: 인접 룸(예: 룸1–로비) 사이 벽의 색/패턴을 바꾸면 반대쪽 룸에서도 바뀌거나, 적용이 불안정함(동시 적용되거나 안 되기도 함).

- **모델:** 벽(wall)은 구조(위치, 길이, 두께, 문/개구부)만 보유. 스타일(색, 패턴, 패턴 강도/반복 크기, 패턴 이미지)은 **벽면(wall face)**에 저장. 모든 벽은 면 2개를 가지며 각 면은 그 면이 향하는 룸 ID에 귀속. 외곽 벽의 바깥 면은 `exterior`로 표기.
- **적용 규칙:** 정면뷰에서 벽 편집 시 "어느 룸에서 진입했는가" 기준으로 해당 룸 쪽 면에만 적용. 우측 패널의 룸 벽 색/패턴 일괄 변경은 **해당 룸을 향한 모든 벽면(인접 룸과의 경계벽 포함)**에 적용되며, 같은 벽의 반대쪽 면(다른 룸 소속)은 절대 영향받지 않음.
  - 예: 룸1의 벽 색을 노란색으로 일괄 변경 → 룸1을 둘러싼 모든 면(룸2와 맞닿은 벽의 룸1 쪽 면 포함)이 노란색. 룸2에 들어가면 룸2에 설정된 색이 그대로 보임.
- **UI:** 정면뷰 상단에 편집 중인 면의 소속 명시(예: "룸 1 — 북쪽 벽 (안쪽 면)"). 반대면 편집 전환 버튼 제공(예: "반대면 편집 → 로비 쪽").
- **일괄 vs 개별:** 룸 일괄 적용 시 개별 지정된 면은 덮어쓰되 undo로 복구 가능하면 충분.
- **마이그레이션:** 기존 프로젝트 파일(벽 단위 스타일) 로드 시 각 벽의 스타일을 양쪽 면에 복사해 호환 유지. 저장 포맷 버전 상향. **뷰어(퍼블리시 결과물)도 면 단위 스타일 렌더링으로 함께 수정.**

**수용 기준:** 룸1과 로비 사이 벽에서 룸1 쪽 면을 딥레드로 변경해도 로비 쪽 면은 기존 색 유지. 에디터 3D 프리뷰와 퍼블리시 결과물 모두에서 동일하게 보임. 기존 프로젝트 파일이 깨지지 않고 열림.

## P4. 텍스트 시스템 개편 (에디터 + 뷰어)

현재 문제: 전시명이 고정 위치(좌상단/문 위)에만 들어가고 줄바꿈 불가. 섹션 소개 텍스트도 위치 고정, 편집이 직관적이지 않음.

- **역할과 표시의 분리:** 전시명/섹션 제목의 데이터 역할(프로젝트 메타데이터, 퍼블리시 시 사이트 제목·로딩 화면 등)은 유지. 벽면 표시는 자유 배치 텍스트 오브젝트로 전환. 역할 없는 **순수 자유 텍스트 오브젝트**도 추가 가능.
- **편집 방식:** 정면뷰에서 작품과 동일한 UX — 드래그로 벽면 어디든 배치(전시명을 문 반대편 벽에 두는 것도 가능), 핸들로 크기 조절, 클릭(또는 더블클릭)으로 내용 수정, **줄바꿈 지원**.
- **폰트:** 기존 명조/고딕에 **Noto Sans KR, Pretendard** 추가. 서브셋 처리 불필요 — 풀 폰트 파일을 퍼블리시 결과물에 포함(용량 증가 허용, URL 배포 전제). 에디터와 뷰어 렌더링 폰트 일치 필수.
- **스타일 옵션:** 굵기, 기울임(이탤릭), 그림자(색/오프셋/블러 간단 프리셋이면 충분), 자간/행간(기존 유지), 색.
- 텍스트 오브젝트도 P7 스마트 가이드, undo 대상에 포함.

**수용 기준:** 전시명을 문 반대편 벽으로 드래그 이동 가능. 두 줄짜리 전시명 입력 가능. 프리텐다드 + 그림자 적용한 텍스트가 에디터 프리뷰와 퍼블리시 결과물에서 동일하게 렌더링됨.

## P5. 뷰어 모바일 최적화

전제: 가상 조이스틱(좌) + 시점 드래그(우) 이동 조작은 이미 정상 동작하므로 건드리지 않음. 이미지 로딩 성능도 현재 문제없음.

- **캐릭터 선택 화면 반응형:** 모바일 세로 화면에서 캐릭터 4개(부루/보리/모아/새롬)가 모두 보이고 선택 가능하도록 레이아웃 수정(2×2 그리드 또는 세로 스크롤 등 화면 폭에 맞게).
- **감상 모드 핀치 줌:** 작품 확대 감상 모드에서 두 손가락 핀치로 줌 인/아웃, 한 손가락 드래그로 패닝. 닫기 버튼 터치 영역 충분히 크게.
- **안내 문구 입력 방식 분기:** 상호작용 프롬프트가 현재 "자세히 보기 E"로 PC 키보드 기준. 터치 기기에서는 "자세히 보기 (터치)" 등 탭 안내로 표시. PC는 기존 "E" 유지. 감지는 포인터 타입 기준(`pointer: coarse` 등).

**수용 기준:** 일반적인 모바일 세로 화면(약 390×844)에서 캐릭터 4개가 스크롤 없이 또는 자연스러운 스크롤로 모두 선택 가능. 감상 모드에서 핀치 줌 동작. 모바일에서 "E" 문구가 보이지 않음.

## P6. 뷰어 카메라 조작 개선

- **수직 시점 제한 완화:** 현재 위(천장) 방향 시점이 막혀 있음. 위쪽 약 80~85도까지 볼 수 있도록 수직 회전 범위 확대(완전 수직은 멀미 방지를 위해 제외). 코퍼드 천장·샹들리에 등 천장 요소가 보여야 함.
- **카메라 거리 줌:** 마우스 휠로 카메라–캐릭터 거리 조절. 휠 당김 = 캐릭터에 가까워짐(1인칭에 근접), 휠 밀기 = 뒤로 빠져 캐릭터와 공간이 넓게 보임. 최소/최대 거리 제한, 벽·오브젝트 뚫림 방지(카메라 충돌 처리). 모바일에서는 **이동 중 두 손가락 핀치**로 동일 기능(감상 모드 핀치 줌과는 모드가 달라 충돌 없음).

**수용 기준:** 고개를 들어 샹들리에가 보임. 휠로 1인칭 근접~넓은 3인칭까지 부드럽게 전환되고 벽에 카메라가 파묻히지 않음. 모바일 이동 중 핀치로 동일 동작.

## P7. 정면뷰 스마트 정렬 가이드 (에디터)

PPT 스마트 가이드와 동일한 UX. 작품·텍스트 오브젝트 드래그 시:

- **벽면 중앙 가이드:** 오브젝트 중심이 벽면의 수평/수직 정중앙에 근접하면 빨간 점선 가이드 표시 + 자석 스냅.
- **오브젝트 간 정렬:** 같은 벽의 다른 오브젝트와 상단/중앙/하단(수직), 좌/중/우(수평) 정렬이 근접하면 가이드 표시 + 스냅.
- **균등 간격:** 3개 이상 나란히 배치 시 간격이 균등해지는 지점에서 간격 가이드(화살표/치수) 표시 + 스냅.
- **스냅 해제:** Alt(⌥) 누른 채 드래그하면 스냅 무시(PPT 관례와 동일).
- 스냅 임계값은 화면 기준 약 6~8px.

**수용 기준:** 작품 하나를 벽 중앙 근처로 끌면 빨간 점선이 뜨며 정중앙에 스냅. 작품 3개를 나란히 놓을 때 균등 간격 지점에서 가이드가 뜨고 스냅. Alt 드래그 시 자유 배치.

## P8. 공통 인프라 (에디터 + 뷰어)

### 8-1. Undo/Redo 전면 적용 (에디터)
이번 패치로 추가되는 모든 조작을 undo/redo 대상에 포함: 룸 이동/크기 조절, 문 위치 이동, 벽면 스타일 변경(면 단위/룸 일괄 모두), 텍스트 오브젝트 생성/이동/크기/내용/스타일, 작품 스냅 배치. 드래그 조작은 드래그 종료 시점 기준 1개 스텝으로 기록.

### 8-2. 저장 안전망 (에디터)
- 미저장 변경이 있을 때 탭 닫기/새로고침 시 브라우저 경고(`beforeunload`).
- 주기적 자동 임시저장(브라우저 로컬, 예: 30초 간격 또는 변경 후 유휴 시). 에디터 재진입 시 임시저장본 복구 여부 확인 다이얼로그.

### 8-3. 관람자 온보딩 오버레이 (뷰어)
- 최초 입장 시 조작법 안내를 1회 표시. PC: 이동(WASD/방향키), 시점(마우스), 감상(E), 줌(휠). 모바일: 조이스틱, 화면 드래그, 작품 탭, 핀치. 입력 방식에 따라 해당 안내만 표시.
- "다시 보지 않기" 반영(로컬 저장). 닫기 쉬운 UI, 3D 로딩을 막지 않게 로딩 완료 후 표시.

### 8-4. 퍼블리시 OG 메타태그 (뷰어)
- 퍼블리시 시 index.html에 Open Graph / Twitter Card 메타태그 자동 삽입: `og:title`(전시명), `og:description`(전시 소개 또는 기본 문구), `og:image`(대표 작품 이미지 — 에디터에서 대표 이미지 지정 가능, 미지정 시 첫 작품), `og:type=website`.
- og:image는 결과물 내 상대 경로가 아닌, 배포 후 절대 URL이 필요하므로 퍼블리시 시 "배포 예정 URL" 입력 필드를 두고 이를 기준으로 절대 경로 생성(미입력 시 상대 경로 유지 + 안내 문구).

**수용 기준:** 룸 드래그 실수 후 Ctrl+Z로 복구. 탭 닫기 시 경고. 처음 접속한 모바일 관람자에게 터치 안내가 1회 표시. 카카오톡에 퍼블리시 URL 공유 시 전시명과 대표 이미지 카드가 표시(배포 URL 입력한 경우).
